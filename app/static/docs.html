API Documentation

Complete reference for the Samsung TV Control Panel REST API. All examples use curl; you can adapt them for your programming language.

GET /api/tvs

List all configured TVs with basic info (no network checks).

Example curl:

curl -X GET "http://localhost:8009/api/tvs"

Example response:

[ { "ip": "192.168.1.100", "name": "T1", "mac": "aa:bb:cc:dd:ee:ff", "token": "abc123...", "ws_port": 8002 } ]
GET /api/tvs/status

Get status of all TVs including online state, power state, and token verification.

Query Parameters:

Parameter Type Default Description
force boolean false Bypass cache and perform fresh checks

Example curl:

curl -X GET "http://localhost:8009/api/tvs/status" curl -X GET "http://localhost:8009/api/tvs/status?force=true"

Example response:

[ { "ip": "192.168.1.100", "name": "T1", "mac": "aa:bb:cc:dd:ee:ff", "online": true, "ping_online": true, "ws_online": true, "token_verified": true, "power_state": "on" } ]
POST /api/tvs/wake-all

Send WOL magic packet to all configured TVs.

Request body:

{ "port": 9 }

Example curl:

curl -X POST "http://localhost:8009/api/tvs/wake-all" \ -H "Content-Type: application/json" \ -d '{"port": 9}'
POST /api/tvs/{ip}/wake

Send WOL magic packet to a specific TV.

Example curl:

curl -X POST "http://localhost:8009/api/tvs/192.168.1.100/wake" \ -H "Content-Type: application/json" \ -d '{"port": 9}'
POST /api/tvs/{ip}/power

Toggle TV power state via WebSocket. If TV is offline, sends WOL first and waits for it to wake.

Example curl:

curl -X POST "http://localhost:8009/api/tvs/192.168.1.100/power"
POST /api/tvs/{ip}/send-key

Send a remote control key command to a specific TV.

Request body:

{ "key": "KEY_POWER" }

Common key codes:

Key Code Description
KEY_POWERPower toggle
KEY_MUTEMute
KEY_VOLUPVolume up
KEY_VOLDOWNVolume down
KEY_UPNavigate up
KEY_DOWNNavigate down
KEY_LEFTNavigate left
KEY_RIGHTNavigate right
KEY_ENTERSelect/OK
KEY_HOMEHome screen
KEY_MENUMenu
KEY_SOURCEInput source
KEY_RETURNBack
KEY_CHUPChannel up
KEY_CHDOWNChannel down

Example curl:

curl -X POST "http://localhost:8009/api/tvs/192.168.1.100/send-key" \ -H "Content-Type: application/json" \ -d '{"key": "KEY_VOLUP"}'
POST /api/tvs/broadcast-key

Send a key command to all TVs simultaneously.

Example curl:

curl -X POST "http://localhost:8009/api/tvs/broadcast-key" \ -H "Content-Type: application/json" \ -d '{"key": "KEY_MUTE"}'
POST /api/tvs/{ip}/token

Request a new authentication token from a TV. TV must be ON and will display a pairing prompt.

Example curl:

curl -X POST "http://localhost:8009/api/tvs/192.168.1.100/token"
POST /api/tvs/refresh-all-tokens

Request new tokens from all TVs that are online and accessible.

Example curl:

curl -X POST "http://localhost:8009/api/tvs/refresh-all-tokens"
GET /api/tvs/{ip}

Get detailed configuration for a specific TV.

Example curl:

curl -X GET "http://localhost:8009/api/tvs/192.168.1.100"
GET /config/tvs.json

Get the raw configuration file (same as /api/tvs but as plain JSON).

Example curl:

curl -X GET "http://localhost:8009/config/tvs.json"
GET /debug/ping

Test if a TV responds to ping.

Query Parameters:

Parameter Type Default Description
ipstringrequiredTV IP address
forcebooleanfalseBypass cache

Example curl:

curl -X GET "http://localhost:8009/debug/ping?ip=192.168.1.100"
GET /debug/port

Check if a TCP port is open on a TV.

Query Parameters:

Parameter Type Default Description
ipstringrequiredTV IP address
portinteger8002Port to check
forcebooleanfalseBypass cache

Example curl:

curl -X GET "http://localhost:8009/debug/port?ip=192.168.1.100&port=8002"
GET /api/tvs/{ip}/ws-check

Verify WebSocket endpoints and token validity for a TV.

Example curl:

curl -X GET "http://localhost:8009/api/tvs/192.168.1.100/ws-check"

Error Handling

All endpoints return standard HTTP status codes and JSON error messages.

{ "detail": "TV not found" }

200 OK — Request succeeded

400 Bad Request — Invalid parameters or missing required fields

404 Not Found — TV or endpoint not found

500 Internal Server Error — Server error; check logs

503 Service Unavailable — Cannot reach the TV or its service

504 Gateway Timeout — Request timed out (TV is slow or unreachable)
